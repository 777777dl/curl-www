First, run these commands: 

  $ mkdir someplacenice
  $ cd someplacenice
  $ cvs -d :pserver:cvsread@cvs.php.net:/repository login
    [enter 'phpfi' (without quotes) as password ]
  $ cvs -d :pserver:cvsread@cvs.php.net:/repository checkout curl
  $ ./curl/testcurl.sh

You'll now see a full compile, build and test perform. Make sure everything works out fine.

Then, in a cron job do: 

 $ cd someplacenice
 $ ./curl/testcurl.sh | mail -s "autobuild" curl-autocompile@haxx.se

If mailing the info is not suitable for you, try using this upload form.

Every once in a while, http://curl.haxx.se/auto/ gets updated with the available logs. Recent CVS changes. SUBTITLE(Build from daily tarballs)

If you can't checkout or update the sources from CVS easily, another approach to automated testing of the latest sources is made by getting the latest daily tarballs automaticly instead.

This is easily done with the use of the autocurl shell script, written by Ralph Mitchell. SUBTITLE(Make more tests run)

The number of tests run on some tests are not the maximum amount, but instead numerous ones are "skipped".

This is because some of the tests require specal conditions that are not enabled everywhere. You can improve this situation by using the configure's --enable-debug option (which enables much more thorough memory and netrc debugging) and to make sure that you have stunnel installed (which enables HTTPS tests to run). #include "foot.html"
