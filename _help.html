The curl project depends on voluntary work by skilled people. We depend on work by many different people for this to develop. Time and resources are scarce for all of us, of course, so if we share the burden we can reach further and create a better product. This list contains some of the stuff we need help with right now.

Have a look at the TODO list we try to maintain!

test suite

  • Fix the perl HTTP server to work with persistent connections
  • Fix the perl FTP server to work with persistent connections.
  • More test cases (especially that exercise persistent connections. We need to improve the whole test suite for this.)
  • add TELNET tests
  • add LDAP tests
misc improvements
  • documentation (all over basically)
  • the new multi interface
  • asynch name resolving
  • zlib support. It should be buffer-based decompression and it should work with the three main compression types HTTP documents can be compressed with.
  • C++ wrappers for the interface

Of course, if you're willing to help and have other ideas for what to improve, fix or add, get in touch and we'll discuss it!
