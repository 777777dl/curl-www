The curl project depend on volountary work by skilled persons. We depend on work by many different people for this to develop. Time and resources are scarce for everyone of us, so if we share the burdens we can reach further and create a better product. This list contains some of the stuff we need help with right now.

Also, have a look at the task list and the TODO list I try to maintain!

test suite

  • Fix the perl HTTP server to work with persistant connections
  • Fix the perl FTP server to work with persistant connections.
  • more test cases (especially that excersise the persistant connections. We might need to improve the whole test suite for this.)
various persistant connections updates
  • Add more example source codes that use persistant connections
misc improvements
  • IPv6 support (passive FTP support is still left)
  • SSL session ID caching
  • asynch name resolving (to allow full win32 timeout support)
  • zlib support. It should be buffer-based decompression and it should work with the three main compression types HTTP documents can be compressed with.
  • C++ wrappers for the interface

Of course, if you're willing to help and have other ideas of what to improve, fix or add, get in touch and we'll discuss it!
