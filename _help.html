The curl project depends on volountary work by skilled people. We depend on work by many different people for this to develop. Time and resources are scarce for all of us, of course, so if we share the burden we can reach further and create a better product. This list contains some of the stuff we need help with right now.

You can have a look at the task list, too, and the TODO list I try to maintain!

web site
The web site is continously growing and getting an extra set of hands to keep things up-to-date and valid would be good...

test suite

  • Fix the perl HTTP server to work with persistent connections
  • Fix the perl FTP server to work with persistent connections.
  • More test cases (especially that exercise persistent connections. We might need to improve the whole test suite for this.)
misc improvements
  • IPv6 support (passive FTP support still has to be done)
  • SSL session ID caching
  • New cookie jar handling
  • Non-blocking connects
  • asynch name resolving (to allow full Win32 timeout support)
  • zlib support. It should be buffer-based decompression and it should work with the three main compression types HTTP documents can be compressed with.
  • C++ wrappers for the interface

Of course, if you're willing to help and have other ideas for what to improve, fix or add, get in touch and we'll discuss it!
