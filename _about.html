This web site and associated mirrors are dedicated to cover curl, libcurl and related issues only.

If you have any complaints, ideas or suggestions for this web site. Don't hesitate to mail feedback to curl-web at haxx.se. We do make a serious effort to please our audience. This email address will always reach the right person.

The curl web site contents are...

  • maintained and edited mostly by Daniel Stenberg. If you wanna join in and help out with this, mail us!
  • banner-less since the VA Linux abandoned their banner program. Mail us to discuss new banners!

The box running curl.haxx.se is...

  • hosted by Contactor Data AB
  • running Apache 1.3.x on a Sun Enterprise 250 machine
  • hooked on a 2 Mbit/sec connection
  • using a subdomain let to us by Haxx
  • serving almost 4000 unique hosts that download about 1GB of data each week.

The main technologies used when creating this site are:

  • Daniel's cpp that all static pages are preprocessed with when generated
  • standard unix make for dependency-stuff
  • perl for the dynamic pages and most background scripting
  • hypermail for all the mail archives
  • LXR for the source indexing magic
  • htdig for the site-local search engine
  • some pages use SSI instructions for dynamic creations
  • Daniel's statistics scripts generate weekly usage summaries
  • man2html by Earl Hood converts the man pages to HTML
  • txt2html by Seth Golub converts some plain-text files to HTML
  • curl(!) for the automaticly generated mirror links
  • contents are managed and distributed using CVS

Hardly any email addresses on this site are displayed plainly with @-letters and all. Research has found that when we publish addresses that way, we receive significantly more spam than when we do it this way. I'm sorry for the inconvenience.

A wide selection of services are provided by sourceforge.net for the benefit of curl users.
