Curl is the tool for getting files with URL syntax, supporting FTP://, HTTP://, HTTPS://, GOPHER://, TELNET://, DICT://, FILE:// and LDAP://. Curl supports HTTPS certificates, HTTP POST, HTTP PUT, FTP uploading, HTTP form based upload, proxies, cookies, user+password authentication and a busload of other useful tricks.

The main use for curl is when you want to get or send files automatically to or from a site using one of the supported protocols.

Curl is free and open software that compiles under a wide variety of operating systems. Curl exists thanks to efforts from many authors, with Daniel Stenberg being the main author and project maintainer. Curl is the result of many spare-time hours of programming. Voluntary contributions are vital for this product! We appreciate your help!

The most recent stable version of curl is version __STABLE, released on __RELDATE. There are currently __CURR available binary archives of the latest curl package, get them from the download page.

If you want to add URL transfer capability to your program, then the libcurl file transfer library might be interesting to you. Libcurl has undergone a reorganization and with version 7 it has really grown to become useful. With your feedback, we can make it even better!

All announcements of new versions will be done at freshmeat.net and to the mailing list.

Report bugs to the mailing list and enter them in the curl bug tracker. The bug system and others are hosted by sourceforge.net, and you can access the entire curl project space here.

Before posting questions to any individual or the mailing list, have a look at the documentation section and the Frequently Asked Questions document.
