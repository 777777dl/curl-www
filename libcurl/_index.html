libcurl is a free and easy-to-use client-side URL transfer library, supporting FTP, FTPS, HTTP, HTTPS, GOPHER, TELNET, DICT, FILE and LDAP. libcurl supports HTTPS certificates, HTTP POST, HTTP PUT, FTP uploading, HTTP form based upload, proxies, cookies, user+password authentication (Basic, Digest, NTLM, Negotiate, Kerberos4), file transfer resume, http proxy tunneling and more!

libcurl is highly portable, it builds and works identically on numerous platforms, including Solaris, NetBSD, FreeBSD, OpenBSD, Darwin, HPUX, IRIX, AIX, Tru64, Linux, UnixWare, HURD, Windows, Amiga, OS/2, BeOs, Mac OS X, Ultrix, QNX, OpenVMS, RISC OS, Novell NetWare, DOS and more...

libcurl is free, thread-safe, IPv6 compatible, feature rich, well supported, fast, thoroughly documented and is already used by many known, big and successful companies and numerous applications.

This whole section of the curl web site is dedicated to libcurl, and it contains detailed information on how to use libcurl in your own applications using your prefered programming language. Checkout the many bindings available in the left-side menu.

Go to the regular curl download page and get the latest curl package. There is no separate package with only libcurl, but there are some binary development packages in the regular curl download section you might opt for. All source and devel packages include libcurl (the binary ones does too, but some of them have it staticly linked into curl). In fact, libcurl is the major part of curl.

Many applications are powered by libcurl and many companies already rely on libcurl for transfers. Check out our using libcurl page for general hints and advice.
