libcurl is a solid and reliable multi-protocol file transfer library for URL syntax defined files. We can take advantage of libcurl straight from within our Perl programs.

All you need to do is to install the Curl::easy package and then you'll be able to use the Perl binding.

How to Install
Install curl and libcurl first. Make sure you install the libcurl.so library and curl's include files as well. (E.g. if you are using an rpm version of curl, you will need to install the curl-devel rpm before you can build the Perl binding.)

Run these commands: 

   cd [curl::easy-directory]
   perl Makefile.PL
   make
   make test
   make install

The Makefile.PL uses curl-config and internal guesses to find curl.h, and to then create a version of Curl::easy which matches your installed libcurl version.

Read and edit Makefile.PL if your curl.h file cannot be found.

Binding documentation
The binding closely resembles the C one, so it should be OK to study the regular libcurl C/C++ API docs for most options.

Perl specific features are described in the Curl::easy POD documentation.

Download
Curl::easy 1.20 (15Kb) was released on February 16, 2002.

This release automatically adapts to the options in your libcurl version, and should support versions from 7.8.1 onwards, including 7.9,7.9.2 thru 7.9.5pre.

Version 1.20 addresses some minor build issues reported with 1.1.9. The version numbering now follows the standard perl module numbering syntax, the same as CPAN modules (major.minor). Support for CURLOPT_STDERR was added.

Curl::easy 1.1.9 (18Kb) was released on January 18, 2002.

This release automatically adapted to the options in your libcurl version, and supports versions from 7.8.1 onwards. Because of minor build issues, use 1.20 instead.

You should upgrade from 1.1.8 if you need access to options in libcurl versions after 7.9.

Curl::easy 1.1.8 (14Kb) was released on September 20, 2001.

This release includes support for libcurl options in libcurl 7.9, though it can also be rebuilt for earlier libcurl versions. Better test scripts are also included.

There is no need to upgrade from 1.1.6 or 1.1.7 unless you want access to new options in libcurl 7.9, such as SSL certificate name checking.

Curl::easy 1.1.7 (12Kb) was released on September 13, 2001.

If you are using curl 7.8.1 or later, you will need to be using at least version 1.1.6 of Curl::easy.

1.1.7 is a documentation only update, with more explicit licence information. There is no need to upgrade if you are already using 1.1.6.
