libcurl is a solid and reliable multi-protocol file transfer library for URL syntax defined files. We can take advantage of libcurl straight from within our Perl programs.

All you need to do is to install the Curl::easy package and then you'll be able to use the Perl binding.

How to Install
Install curl and libcurl first. Make sure you install the libcurl.so library and curl's include files as well. (E.g. if you are using an rpm version of curl, you will need to install the curl-devel rpm before you can build the Perl binding.)

Run these commands: 

   cd [curl::easy-directory]
   perl Makefile.PL
   make
   make test
   make install

The Makefile.PL uses curl-config and internal guesses to find curl.h, and to then create a version of Curl::easy which matches your installed libcurl version.

Read and edit Makefile.PL if your curl.h file cannot be found.

Binding documentation
The binding closely resembles the C one, so it should be OK to study the regular libcurl C/C++ API docs for most options.

Perl specific features are described in the Curl::easy POD documentation.

This release automatically adapts to the options in your libcurl version, and should support versions from 7.8.1 onwards, including 7.9,7.9.2 thru 7.9.8

Version 1.21 integrates some minor build changes to assist in building on Win32, after feedback from Shawn Poulson. There are no functional changes

This is the 'best' available version of Curl::easy, and should be used for all versions of curl from 7.8.1 onwards.

Version 1.20 addressed some minor build issues reported with 1.1.9. The version numbering now follows the standard perl module numbering syntax, the same as CPAN modules (major.minor). Support for CURLOPT_STDERR was added.
