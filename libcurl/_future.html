We're starting to talk and plan for a future curl and libcurl version 8.

Everything in the crrent libcurl is now subject for discussions around what to keep, what to improve, what to add and what to remove. If you object to any of the points below, make your voice heard and tell us your view!

SUBTITLE(subject for removal in v8)

  • DICT support

SUBTITLE(subject for improvement in v8)

  • LDAP support.
SUBTITLE(subject for addition in v8)

  • the multi interface
  • NTLM support
  • shareable connection pool
  • DNS caching
  • asynch DNS lookups
TITLE(multi API)

We intend to add a new interface to libcurl, an interface that allows the calling application to "drive". We call that a "pull" interface.

This was once upon the time dicussed, calling it a "low level" interface. It was brought up again during the autumn 2001 and we have now refined our ideas and our thoughts are now gathered around the "multi" interface.

This interface makes it very easy to transfer multiple files simultaneously, while also allowing the application to do stuff in the mean time. All this without using multiple threads. SUBTITLE(header and initial source code)

We have started working on the multi.h header file, which also pretty much contains all the docs there is yet. The multi.c file is to be filled in with (more) source code. SUBTITLE(example using this interface)

The imaginary multi-app.c is application source code, showing how this interface could be used.
